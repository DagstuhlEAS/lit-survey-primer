Goal

A literature survey is a systematic qualitative and/or quantitative analysis of publications in a particular field/discipline with the aim of understanding something about that field/discipline. There are many such studies, and this is intended to help make them relatively more comparable across sampling populations and over time.

This documents provides (1) steps on running a literature survey to asses the accessibility of code in an academic publication, and (2) examples and learnings from other such surveys. The steps can (and should) be amended, and the examples expanded.

This a living document that welcomes contributions, therefore it is hosted on GitHub. However, recognizing that not everyone who can make an important contribution is familiar with GitHub, please add any contributions as comments on this issue [LINK TBA]

Steps for running a literature survey

  1. Identify a sample. Example: 3 different publications from 3 different publishers, for a defined period of time.
    Key challenges:
    1. Articulate what contitutes code that "ought" to be made available.
    2. Articulate how sample relates to broader literature, and how (and whether) the forthcoming findings can be generalized.
    3. Document with precision the inclusion criterion.
  2. Categorize whether the code that ought to be available is.
    The @asclnet approach consist of sending personal request emails for code access, and then categorizing by whether the code was (1) available without email, (2) available upon email, possibly with some coaching, and (3) no response / not available.
